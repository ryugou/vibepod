# v1.4.0 実装計画

## 概要

v1.4.0 では以下の4機能を追加する。

1. バージョン表示（バナー直下）
2. `--worktree` オプション（worktree 隔離実行）
3. 実行後の diff サマリー表示
4. `--lang` オプション + 自動検出（言語ツールチェインのインストール）

## 2. `--worktree` オプション

### 変更対象

- `src/cli/mod.rs` — CLI 引数定義に `--worktree` 追加
- `src/main.rs` — `worktree` 引数を `execute` に渡す
- `src/cli/run.rs` — worktree 作成・マウント・後処理ロジック

### CLI 引数

```rust
/// Run in an isolated git worktree (for --prompt mode)
#[arg(long)]
worktree: bool,
```

`--worktree` は `--prompt` と併用する前提。`--prompt` なしで `--worktree` が指定された場合はエラーにする。

### 動作フロー

1. `--worktree` が指定されている場合：
   - 一時的なブランチ名を生成（`vibepod/prompt-<timestamp>`）
   - `git worktree add .worktrees/<branch-name> -b <branch-name>` で worktree を作成
   - worktree のパスをコンテナの `/workspace` としてマウント（元の cwd の代わり）
   - コンテナ実行
   - 完了後に diff サマリーを表示（→ 機能3）
   - worktree のパスとブランチ名を表示して終了（worktree は削除しない。ユーザーが確認してからマージ/削除する）

2. `--worktree` が指定されていない場合：
   - 現行通りの動作

### 出力イメージ

起動時：
```
  ◇  Created worktree: .worktrees/vibepod-prompt-20260402-143000
  │  Branch: vibepod/prompt-20260402-143000
```

完了時：
```
  ◇  Worktree diff summary:
  │   src/ui/banner.rs | 5 +++--
  │   src/cli/run.rs   | 3 ++-
  │   2 files changed, 5 insertions(+), 3 deletions(-)
  │
  ◇  Worktree: .worktrees/vibepod-prompt-20260402-143000
  │  Branch: vibepod/prompt-20260402-143000
  │  To review: cd .worktrees/vibepod-prompt-20260402-143000 && git diff main
  │  To merge:  git merge vibepod/prompt-20260402-143000
  │  To remove: git worktree remove .worktrees/vibepod-prompt-20260402-143000
```

## 3. 実行後の diff サマリー表示

### 変更対象

- `src/cli/run.rs` — コンテナ停止後の処理

### 動作

コンテナ停止後に、マウントしていたワークスペースで `git diff --stat` を実行して結果を表示する。

- `--worktree` モード: worktree 内で `git diff --stat` を実行
- `--worktree` なしの `--prompt` モード: cwd で `git diff --stat` を実行
- interactive / resume モード: 表示しない（ユーザーが直接操作しているため）

### 実装

```rust
// コンテナ停止後
if prompt.is_some() {
    let diff_dir = if worktree { &worktree_path } else { &cwd };
    let output = Command::new("git")
        .args(["diff", "--stat"])
        .current_dir(diff_dir)
        .output()?;
    let stat = String::from_utf8_lossy(&output.stdout);
    if !stat.trim().is_empty() {
        println!("  ◇  Changes:");
        for line in stat.lines() {
            println!("  │  {}", line);
        }
    }
}
```

## 4. `--lang` オプション + 自動検出

### 変更対象

- `src/cli/mod.rs` — CLI 引数定義に `--lang` 追加
- `src/main.rs` — `lang` 引数を `execute` に渡す
- `src/cli/run.rs` — 言語検出・セットアップスクリプト生成
- `src/runtime/docker.rs` — ContainerConfig にセットアップコマンドを追加

### CLI 引数

```rust
/// Language toolchain to install in container (rust, node, python, go, java)
#[arg(long)]
lang: Option<String>,
```

### 自動検出ロジック

`--lang` が未指定の場合、マウントするワークスペース内のファイルから検出する：

| ファイル | 言語 |
|---------|------|
| `Cargo.toml` | rust |
| `package.json` | node |
| `go.mod` | go |
| `pyproject.toml`, `requirements.txt` | python |
| `pom.xml`, `build.gradle`, `build.gradle.kts` | java |

複数言語が検出された場合、すべてインストールする。

### インストール方法

コンテナの entrypoint を変更して、claude 起動前にセットアップコマンドを実行する。
現在の args は `["claude", ...]` だが、これを `["sh", "-c", "<setup> && claude ..."]` に変更する。

各言語のインストールコマンド（コンテナは Debian ベース）：

| 言語 | コマンド |
|------|---------|
| rust | `apt-get update && apt-get install -y build-essential && curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs \| sh -s -- -y && . $HOME/.cargo/env` |
| node | `curl -fsSL https://deb.nodesource.com/setup_22.x \| bash - && apt-get install -y nodejs` |
| python | `apt-get update && apt-get install -y python3 python3-pip python3-venv` |
| go | `curl -fsSL https://go.dev/dl/go1.24.2.linux-arm64.tar.gz \| tar -C /usr/local -xzf - && export PATH=$PATH:/usr/local/go/bin` |
| java | `apt-get update && apt-get install -y default-jdk` |

注意: go のダウンロード URL はアーキテクチャ依存（arm64/amd64）。ランタイムで `uname -m` を見て切り替える。

1. バージョン表示（最も小さい変更）
2. `--lang` + 自動検出（コンテナ起動部分の変更）
3. `--worktree`（ワークフロー変更）
4. diff サマリー（3 の上に乗せる）
